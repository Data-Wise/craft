    Progress Tracking¶

    Choice: Hybrid auto-calculate with manual override

    Rationale: - Auto-calculation from dates (95% accurate) - Manual override for edge cases (guest lectures, makeup classes) - Stored in teach-config.yml for visibility

    Implementation: 

    def get_current_week(config, override=None):
    if override:
        return override

    # Auto-calculate
    start = datetime.fromisoformat(config['dates']['start'])
    today = datetime.now()

    # Account for breaks
    breaks = config['dates']['breaks']
    days_on_break = sum(break_days(b) for b in breaks if in_past(b))

    days_since_start = (today - start).days - days_on_break
    week = (days_since_start // 7) + 1

    return week

    Implementation Blockers¶

    Blocker 1: Quarto Multi-Branch Publishing¶

    Issue: Quarto's quarto publish gh-pages hardcoded to single branch

    Workaround: - Use manual git merge + git push workflow - Configure GitHub Pages to use branch selector - Craft manages the merge, GitHub Pages serves result

    Resolution: Not a blocker, use git-native workflow

    Blocker 2: Validation Performance¶

    Issue: Link checking slow on large sites (100+ pages)

    Options: - Cache validation results - Incremental checks (only changed files) - Background validation during idle time

    Resolution: Start with full validation, optimize later

    Blocker 3: Config File Adoption¶

    Issue: Users may not want to create .flow/teach-config.yml

    Fallback: - Detect from _quarto.yml metadata - Infer from project structure - Prompt to create config on first teaching command

    Resolution: Multiple detection strategies (already planned)
