    Proposal: Brainstorm Question Control¶

    Created: 2026-01-18 Updated: 2026-01-18 Status: ✅ Phase 1 Ready to Implement | 📋 Phase 2/3 Future Enhancements Request: Custom question count + question categories

    Executive Summary¶

    Aspect Detail
    Problem No fine-grained control over brainstorm questions (0, 2, or 8 only)
    Solution Colon notation d:5, unlimited questions, category filtering
    MVP (Phase 1) ✅ Ready - 12 hours, ships in v2.4.0
    Future (Phase 2/3) 📋 Deferred - 24 hours, only if validated by usage
    Risk Mitigation Ship MVP first, gather feedback, avoid over-engineering

    Decision: Implement Phase 1 only. Evaluate Phase 2/3 after 2-4 weeks of real usage.

    Overview¶

    Enhance /workflow:brainstorm to give users control over:

    1. Number of questions asked during brainstorming
    2. Categories of questions to focus on

    Current Behavior¶

    Depth Questions Categories
    quick 0 None
    default 2 Auto-selected
    deep 8 All categories
    max 8 + agents All + agent analysis

    Problem: No fine-grained control. Users must choose between 0, 2, or 8 questions.

    Proposed Solution¶

    Implementation Strategy (Impact-Based Phasing)¶

    Phase Impact Hours Features Status
    1 ⭐ High 12 Colon notation, unlimited, categories, question bank ✅ Ready to Implement
    2 📊 Medium 9 Question styles, validation, saved prefs 📋 Future Enhancement
    3 🎁 Nice 15 Custom questions, templates, session memory 📋 Future Enhancement

    Strategy: Ship Phase 1 (MVP) first → Get feedback → Decide on Phase 2/3

    ✅ Phase 1: Ready to Implement (v2.4.0)¶

    Scope: Core question control features - 12 hours

    What's Included:

    • ✅ Colon notation parsing (d:5, m:12, q:3)
    • ✅ Unlimited questions with milestone prompts (every 8 questions)
    • ✅ Categories flag (--categories or -C)
    • ✅ Question bank (8 categories × 2 questions each)
    • ✅ Documentation updates
    • ✅ Tests (unit + integration)

    Deliverable: Fully functional MVP that addresses the original request

    📋 Phase 2/3: Future Enhancements¶

    Deferred pending Phase 1 feedback and usage validation

    Phase 2 (UX Polish) - 9 hours:

    • Question style choice (batch/one-at-a-time/all-upfront)
    • Validation & error recovery prompts
    • Saved preferences (.flow/brainstorm-prefs.yml)

    Phase 3 (Power Users) - 15 hours:

    • Custom questions per project
    • Project type templates
    • Session memory and context reuse

    Decision Criteria:

    • Ship Phase 1 → Monitor usage for 2-4 weeks
    • Gather user feedback on pain points
    • Only implement Phase 2/3 if users request these features

    New Arguments¶

    /brainstorm [depth:N] [focus] [action] "topic" [--categories CAT1,CAT2]
            ↑                                   ↑
            │                                   └── Filter question types
            └──────────────────────────────────── Depth with optional question count

    Argument: depth:N (Colon Notation)¶

    Syntax: depth:count where depth is d|deep|m|max|q|quick|default and count is a number.

    Behavior: Override the default question count for the selected depth.

    Syntax Depth Mode Questions Description
    d deep 8 (default) Standard deep mode
    d:5 deep 5 Deep mode with exactly 5 questions
    d:12 deep 12 Deep mode with 12 questions
    m:20 max 20 Max mode with 20 questions
    q:2 quick 2 Quick mode but ask 2 questions
    default:4 default 4 Default mode with 4 questions

    Important Rules:

    1. Depth required - Must specify depth when using count (e.g., d:5, not just :5)
    2. Override not add - Count replaces depth default (d:5 = exactly 5, not 8+5)
    3. No upper limit - Any count allowed, continuation offered at milestones
    4. Zero allowed - d:0 is valid (deep mode analysis, no questions)

    Examples:

    # Basic usage
/brainstorm d:5 "auth"                  # Deep mode, exactly 5 questions
/brainstorm m:12 f s "api"              # Max, 12 questions, feature, save spec
/brainstorm q:3 "notifications"         # Quick mode but ask 3 questions

# Full combinations
/brainstorm d:6 f "auth"                # Deep:6, feature focus
/brainstorm m:20 a s "microservices"    # Max:20, arch focus, save spec
/brainstorm default:10 "caching"        # Default mode, 10 questions

    Validation & Error Handling:

    When invalid syntax detected, prompt to clarify:

    User: /brainstorm d:abc "auth"

AskUserQuestion:
  question: "Invalid question count 'abc'. What did you mean?"
  header: "Count"
  multiSelect: false
  options:
    - label: "Use deep default (8 questions)"
      description: "Standard deep mode"
    - label: "0 questions"
      description: "Skip questions, just analyze"
    - label: "Enter number"
      description: "Specify exact count"

    Special cases:

    • d:0 → Deep analysis with zero questions (relies on context)
    • m:0 → Max mode with agents, no questions
    • Invalid: :5 alone → Prompts for depth selection

    Argument: --categories (or -C)¶

    Select which question categories to include.

    Question Categories¶

    Category Short Questions Asked
    requirements req What are the key requirements? What constraints exist?
    users usr Who are the users? What are their needs?
    scope scp What's in/out of scope? MVP vs nice-to-have?
    technical tech Technical constraints? Architecture preferences?
    timeline time Deadlines? Milestones?
    risks risk What could go wrong? Edge cases?
    existing exist What already exists? Dependencies?
    success ok How will we measure success? Acceptance criteria?

    Full Question Bank (by category):

    requirements:
  - "What are the key requirements for this feature?"
  - "Are there any hard constraints we must work within?"

users:
  - "Who is the primary user for this feature?"
  - "What problem does this solve for them?"

scope:
  - "What's definitely in scope for the first iteration?"
  - "What's explicitly out of scope (nice-to-have later)?"

technical:
  - "Are there technical constraints or preferences (languages, frameworks)?"
  - "What existing systems does this need to integrate with?"

timeline:
  - "Are there any deadlines or milestones?"
  - "Is there a target date for first working version?"

risks:
  - "What could go wrong? What are the biggest risks?"
  - "Are there edge cases we should plan for upfront?"

existing:
  - "What existing code/systems can we leverage?"
  - "What dependencies does this have?"

success:
  - "How will we know this is successful?"
  - "What are the acceptance criteria?"

    Usage:

    # Ask only requirements and scope questions
/brainstorm "auth" --categories req,scp

# Technical + risks only
/brainstorm "api design" -C tech,risk

# All categories (default for deep)
/brainstorm d "microservices" --categories all

    Combining with --questions:

    # 4 questions from requirements and users categories
/brainstorm "auth" --questions 4 --categories req,usr

# 2 questions each from 3 categories (6 total)
/brainstorm "api" -Q 6 -C req,tech,success

    Argument Parsing (Extended)¶

    # Add to existing DEPTH_MAP, FOCUS_MAP, ACTION_MAP
CATEGORY_MAP = {

def parse_brainstorm_args(args):
    """Parse brainstorm arguments including new --questions and --categories."""
    result = {
        'depth': 'default',
        'focus': 'auto',
        'action': None,
        'topic': None,
        'questions': None,  # None = use depth default
        'categories': None  # None = all categories
    }

    # Parse --questions N or -Q N
    if '--questions' in args or '-Q' in args:
        result['questions'] = extract_number(args)

    # Parse --categories or -C
    if '--categories' in args or '-C' in args:
        result['categories'] = extract_categories(args)

    return result

    Question Selection Logic¶

    def select_questions(depth, question_count, categories):
    """Select questions based on user preferences."""

    # Get all questions from selected categories
    available = []
    for cat in categories or ALL_CATEGORIES:
        available.extend(QUESTION_BANK[cat])

    # Determine count
    if question_count is not None:
        count = question_count
    else:
        count = DEPTH_DEFAULTS[depth]  # 0, 2, or 8

    # Select questions (prioritize by importance if count < available)
    if count >= len(available):
        return available
    else:
        return prioritize_questions(available, count)

def prioritize_questions(questions, count):
    """Select most important questions when limiting count."""
    # Priority order: requirements > users > scope > technical > success
    priority_order = ['requirements', 'users', 'scope', 'technical',
                      'success', 'risks', 'timeline', 'existing']

    selected = []
    for cat in priority_order:
        if len(selected) >= count:
            break
        cat_questions = [q for q in questions if q['category'] == cat]
        selected.extend(cat_questions[:max(1, count // len(priority_order))])

    return selected[:count]

    Enhanced Features (Based on Feedback)¶

    Feature 1: Unlimited Questions with Continuation Prompts¶

    Behavior: No hard upper limit. Continuation prompts offered at milestones.

    Syntax Questions Continuation Prompt
    q 0 (default) After 0: "Ask questions or start?"
    q:5 5 After 5: "Continue or start?"
    d 8 (default) After 8: "Continue or start?"
    d:12 12 After 8: "Continue (4 more)?" → After 12: "Continue or start?"
    d:20 20 Prompts every 8 questions
    m:30 30 Prompts every 8 questions + agent check

    Milestone Pattern:

    • Every 8 questions → Continuation prompt
    • User can request: +4 more, +8 more, or "Keep going"
    • "Keep going" mode → Prompts every 4 questions until user says done

    Continuation Flow:

    Claude: You've answered 8 questions. Good context so far!

AskUserQuestion:
  question: "Continue gathering context or start brainstorming?"
  header: "Continue"
  multiSelect: false
  options:
    - label: "Done - Start brainstorming (Recommended)"
      description: "Proceed with current context"
    - label: "4 more questions"
      description: "Add a few more details"
    - label: "8 more questions"
      description: "Thorough exploration"
    - label: "Keep going until I say stop"
      description: "I'll tell you when I'm done"

    "Keep going" mode:

    • Asks questions indefinitely
    • After every 4 questions: "Continue? [Yes/Done]"
    • User says "done" or selects Done option to proceed

    Decision Log¶

    Decision Choice Rationale
    Syntax style Colon notation (d:5, m:12) Consistent with shorthand style, not verbose
    Question count behavior Override not add d:5 = exactly 5, not 8+5 (clearer)
    Standalone count Require explicit depth :5 alone → error/prompt (prevents confusion)
    Max question limit No upper limit Unlimited with milestone prompts (ADHD-friendly)
    Invalid syntax handling Prompt to clarify Interactive recovery vs hard error
    Categories syntax Keep as flag (--categories) Different purpose than depth/count
    Priority categories req, users, scope, existing User feedback from questions
    Question style User choice per session Flexibility over rigidity
    Persistence location .flow/ directory Consistent with other craft state
    Custom questions YAML config + command Both easy and scriptable
