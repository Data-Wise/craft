    Test Approach¶

    Static Analysis¶

    The test suite uses static analysis rather than subprocess execution:

    Benefits: - ‚úÖ More reliable in test environments - ‚úÖ No subprocess/environment issues - ‚úÖ Fast execution (< 25ms total) - ‚úÖ Validates all critical logic - ‚úÖ No external dependencies

    Techniques: - Content parsing (YAML frontmatter, bash scripts) - Syntax validation (bash -n for shell scripts) - Pattern matching (regex for logic detection) - Structure validation (file existence, permissions)

    Example (Agent Hooks): 

    def test_hook_handles_pretooluse_event(self):
    """Hook should have PreToolUse event handler."""
    content = self.get_hook_content()

    assert 'PreToolUse)' in content, \
        "Hook should have PreToolUse case handler"

    # Should check resource limits
    assert 'ACTIVE_AGENTS' in content, \
        "PreToolUse handler should check active agents"

    Execution Testing¶

    Full execution testing is deferred to E2E tests: - ‚úÖ Unit tests: Structure, syntax, logic (DONE) - üîú E2E tests: Runtime behavior, integration - üîú Dogfooding: Real-world workflows
