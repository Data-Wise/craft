# Troubleshooting: CI Workflow Fails

**Level:** Beginner

## Problem

A GitHub Actions workflow generated by Craft fails in CI:

```
Run npx markdownlint-cli2 "**/*.md"
Error: Cannot find module 'markdownlint-cli2'
```

## Common Causes & Solutions

### 1. Missing Dependencies in CI

**Issue:** The workflow references tools not installed in the CI runner.

**Solution:** Add installation steps before the command:

```yaml
- name: Install dependencies
  run: npm install -g markdownlint-cli2
- name: Lint markdown
  run: npx markdownlint-cli2 "**/*.md"
```

**Why:** CI runners start clean. Local dependencies must be explicitly installed.

### 2. Wrong Node or Python Version

**Issue:** CI uses a different runtime version than local.

**Solution:** Pin versions in the workflow:

```yaml
- uses: actions/setup-node@v4
  with:
    node-version: '20'
- uses: actions/setup-python@v5
  with:
    python-version: '3.11'
```

### 3. Secrets Not Configured

**Issue:** Workflow references secrets not set in the repository.

```
Error: Input required and not supplied: token
```

**Solution:** Go to repo Settings > Secrets and variables > Actions and add the required secrets. Verify names match the workflow YAML.

### 4. YAML Syntax Errors

**Issue:** Invalid YAML that GitHub rejects.

```
Error: mapping values are not allowed in this context
```

**Solution:** Validate before pushing:

```bash
/craft:ci:validate
```

Common pitfalls: tabs instead of spaces, unquoted colons in values, missing block scalar indicators.

### 5. File Not Found or Not Executable

**Issue:** Workflow references missing files or files without execute permission.

**Solution:** Verify tracked (`git ls-files`) and make executable (`git update-index --chmod=+x`).

## Verification Steps

```bash
/craft:ci:validate                        # Validate workflow YAML
npx markdownlint-cli2 "**/*.md"          # Run checks locally
git push && gh run list --limit 1         # Push and check run
```
