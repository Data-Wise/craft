# SPEC: CLAUDE.md Optimization & Command Reform v3.0

**Status:** draft

**From Brainstorm:** BRAINSTORM-claude-md-optimization-2026-02-04.md

## Overview

Redesign the CLAUDE.md command suite to enforce token-efficient CLAUDE.md files using a pointer/trigger architecture. Reduce 5 commands to 3 (init, sync, edit). Add iA Writer integration for external editing with sync-back. Enforce a hard < 150 line budget with pre-commit hook enforcement.

## Primary User Story

**As a** developer using Claude Code,
**I want** my CLAUDE.md to be as lean as possible (< 150 lines),
**so that** every session starts with minimal token overhead while still having access to full project documentation on-demand via pointers.

### Acceptance Criteria

- [ ] CLAUDE.md files generated by `init` are < 150 lines
- [ ] `sync` command refuses to add release notes (hard block)
- [ ] `sync --optimize` moves bloated sections to detail files
- [ ] `edit` command opens iA Writer with TODO annotations
- [ ] Pre-commit hook blocks commits where CLAUDE.md > 150 lines
- [ ] Pointer format: `-> Full history: [VERSION-HISTORY.md](docs/VERSION-HISTORY.md)`
- [ ] All 4 standard detail files supported (VERSION-HISTORY, ARCHITECTURE, CONTRIBUTING, COMMANDS)
- [ ] `--global` flag works on ~/.claude/CLAUDE.md

## Secondary User Stories

**As a** craft plugin maintainer,
**I want** the `sync` command to auto-audit after updating metrics,
**so that** I catch issues in a single command invocation.

**As a** developer working across multiple projects,
**I want** the global CLAUDE.md to use pointers to project-specific docs,
**so that** project-specific detail doesn't pollute every session.

## Pointer Architecture

### Format Standard

Pointer lines use markdown link with arrow prefix:

```markdown
-> Full release history: [VERSION-HISTORY.md](docs/VERSION-HISTORY.md)
```

Claude Code naturally understands markdown links and will `Read` the target file when the user asks about that topic. The arrow prefix (`->`) makes pointers visually distinct from regular links.

### Standard Detail Files

| Detail File | Contains | Pointer In CLAUDE.md |
|-------------|----------|---------------------|
| `docs/VERSION-HISTORY.md` | All release notes, changelogs, migration guides | `-> Release history: [VERSION-HISTORY.md](docs/VERSION-HISTORY.md)` |
| `docs/ARCHITECTURE.md` | Detailed system design, component diagrams, data flows | `-> Architecture detail: [ARCHITECTURE.md](docs/ARCHITECTURE.md)` |
| `CONTRIBUTING.md` | Dev setup, testing guide, PR process details | `-> Contributing guide: [CONTRIBUTING.md](CONTRIBUTING.md)` |
| `docs/COMMANDS.md` | Full command/API reference with all options and examples | `-> Command reference: [COMMANDS.md](docs/COMMANDS.md)` |

### P0 Sections (Always in CLAUDE.md)

These sections are never cut, regardless of budget:

| Section | Max Lines | Content |
|---------|-----------|---------|
| **Header/TL;DR** | 5 | Name, version, one-liner, key stats |
| **Git Workflow** | 15 | Branch strategy, commit conventions, worktree commands |
| **Quick Commands** | 15 | Build, test, lint, validate — the daily-use commands |
| **Project Structure** | 15 | Directory layout with key files (tree format) |
| **Troubleshooting** | 10 | Top 5-8 most common issues and fixes |
| **Pointers** | 10 | Links to detail files |
| **Total P0** | **~70** | Leaves 80 lines for P1 content |

### P1 Sections (Include if under budget)

| Section | Max Lines | Content |
|---------|-----------|---------|
| **Agents/Routing** | 10 | Agent table for task routing |
| **Execution Modes** | 8 | Mode table (default, debug, optimize, release) |
| **Active Development** | 5 | Current branch + version only |
| **Key Files** | 15 | Top 15 most-referenced files |

### P2 Sections (Always in detail files, never in CLAUDE.md)

| Content | Detail File |
|---------|-------------|
| Release notes/history | VERSION-HISTORY.md |
| Feature matrices | VERSION-HISTORY.md |
| Per-test-file breakdowns | CONTRIBUTING.md |
| Phase completion details | VERSION-HISTORY.md |
| PR/merge details | VERSION-HISTORY.md |
| Diffstats (+lines/-lines) | Never documented |
| Implementation phase notes | Never documented (delete after release) |

### Anti-Pattern Rules (Hard Blocks)

The `sync` command **refuses** to add these to CLAUDE.md:

| Pattern | Detection | Action |
|---------|-----------|--------|
| Release notes sections | Regex: `^###? v\d+\.\d+` or `Released \d{4}` | Block + redirect to VERSION-HISTORY.md |
| Diffstats | Regex: `Files Changed:.*\+.*/-` | Block + delete |
| "What Shipped" sections | Regex: `What Shipped\|Merged PR` | Block + redirect |
| Completed feature tables | Regex: `Status.*Complete\|Released ✅` | Block + redirect |
| Phase implementation details | Regex: `^### Phase \d` with completion markers | Block + redirect |

## iA Writer Edit Workflow

### Sequence Diagram

```
User                    Claude                   iA Writer
  │                       │                         │
  ├── /claude-md:edit ──> │                         │
  │                       ├── Read CLAUDE.md        │
  │                       ├── Analyze sections      │
  │                       ├── Insert TODO hints     │
  │                       ├── Write annotated file  │
  │                       ├── AppleScript open ───> │
  │                       │                         ├── File opens
  │                       ├── "Editing in iA Writer"│
  │                       ├── "Type 'done' when     │
  │                       │    finished"            │
  │   (edit in iA Writer) │                         │
  │                       │                    save │
  ├── "done" ───────────> │                         │
  │                       ├── Re-read CLAUDE.md     │
  │                       ├── Remove TODO comments  │
  │                       ├── Run audit             │
  │                       ├── Show validation report│
  │                       │                         │
  │  <── Report ──────────┤                         │
```

### TODO Hint Format

Before opening, Claude inserts HTML comments with optimization guidance:

```markdown
<!-- TODO: Section "Recent Major Features" is 174 lines. Target: 0 lines.
     Action: Move to docs/VERSION-HISTORY.md, replace with pointer. -->
## Recent Major Features

<!-- TODO: Section "Test Suite" is 25 lines. Target: 8 lines.
     Action: Keep run command only, remove per-file breakdown. -->
## Test Suite
```

### AppleScript Integration

```bash
# Open CLAUDE.md in iA Writer
osascript -e 'tell application "iA Writer"
    activate
    open POSIX file "/path/to/CLAUDE.md"
end tell'
```

### Post-Edit Validation

After user says "done":

1. Re-read the file
2. Strip all `<!-- TODO: ... -->` comments
3. Run full audit (5 checks)
4. Check line count against budget
5. Report results

```
Edit Complete - Validation Report

Lines: 142 (budget: 150) ✅
Sections: 8 present ✅

Audit Results:
  ✅ Version matches source
  ✅ All commands documented
  ✅ Required sections present
  ✅ No stale references
  ⚠️ 1 broken link (line 89: docs/old-guide.md)

Fix broken link? [y/n]
```

## Pre-Commit Hook

### Hook Configuration

Add to `.pre-commit-config.yaml`:

```yaml
- repo: local
  hooks:
    - id: claude-md-budget
      name: CLAUDE.md line budget
      entry: bash -c 'lines=$(wc -l < CLAUDE.md 2>/dev/null || echo 0); if [ "$lines" -gt 150 ]; then echo "CLAUDE.md is $lines lines (budget: 150). Run /craft:docs:claude-md:sync --optimize"; exit 1; fi'
      language: system
      files: CLAUDE.md
      pass_filenames: false
```

### CI Integration

Add to `docs-quality.yml`:

```yaml
- name: Check CLAUDE.md budget
  run: |
    lines=$(wc -l < CLAUDE.md)
    if [ "$lines" -gt 150 ]; then
      echo "::error::CLAUDE.md is $lines lines (budget: 150)"
      exit 1
    fi
```
